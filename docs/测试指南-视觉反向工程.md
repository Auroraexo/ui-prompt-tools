# 视觉反向工程功能测试指南

## 功能概述

"视觉反向工程"功能允许用户上传 UI 截图，AI 自动提取样式参数（圆角、间距、配色、阴影等）并回填到左侧配置表单。

## 测试步骤

### 1️⃣ 启动服务

```bash
# 启动后端 AI 服务
npm run dev:server

# 另开一个终端，启动前端
npm run dev
```

确保环境变量 `ARK_API_KEY` 已配置。

### 2️⃣ 准备测试截图

建议准备以下类型的 UI 截图进行测试：

#### 测试用例 A：现代毛玻璃风格
- 特征：圆润按钮（`rounded-3xl`）、毛玻璃背景、柔和阴影
- 预期提取：`bg-white/5 backdrop-blur-xl`、`rounded-2xl`、`shadow-[...]`

#### 测试用例 B：极简扁平风格
- 特征：直角或小圆角、纯色背景、无阴影
- 预期提取：`rounded-md` 或 `rounded-none`、`bg-white`、无明显阴影

#### 测试用例 C：渐变科技风
- 特征：大圆角、渐变背景、彩色阴影
- 预期提取：`rounded-3xl`、渐变色类名、彩色阴影值

#### 测试用例 D：深色主题
- 特征：深色背景、高对比文字、半透明元素
- 预期提取：`bg-zinc-900`、`text-white/90`、透明度相关类名

### 3️⃣ 执行测试

1. **上传截图**
   - 方式 1：拖拽本地截图到"上传本地图片"输入框
   - 方式 2：输入公网图片 URL（推荐使用 Imgur、GitHub 等）

2. **点击"✨ 提取参数"按钮**
   - 观察按钮状态变为"提取中..."
   - 等待 AI 返回结果（通常 5-15 秒）

3. **验证提取结果**
   - 检查绿色提示框中是否显示"✅ 已成功提取并应用 UI 参数！"
   - 查看左侧配置区域，各维度的参数是否已更新
   - 查看右侧 Prompt 区域，生成的提示词是否包含新参数

4. **微调与验证**
   - 手动调整左侧某些参数（如圆角大小）
   - 验证右侧 Prompt 实时更新
   - 点击"复制"按钮，粘贴到 Cursor 验证效果

### 4️⃣ 异常情况测试

#### 测试场景 1：无效图片
- 上传非图片文件（如 PDF、文本文件）
- **预期**：前端拦截，提示"仅支持图片格式"

#### 测试场景 2：超大图片
- 上传 10MB+ 的高清截图
- **预期**：前端警告或后端返回 413 错误，提示"请求体过大"

#### 测试场景 3：无法识别的 UI 风格
- 上传非常抽象的艺术图片或纯文字截图
- **预期**：AI 返回默认推荐值，或提示"无法准确提取"

#### 测试场景 4：网络超时
- 断开网络或使用慢速连接
- **预期**：60 秒后提示"请求超时"

### 5️⃣ 验收标准

✅ **基础功能**
- [ ] 能成功上传截图（本地文件 + URL）
- [ ] 点击"提取参数"后按钮禁用并显示加载状态
- [ ] AI 返回结果后左侧配置自动更新
- [ ] 右侧 Prompt 同步更新

✅ **准确性**
- [ ] 圆角提取基本准确（误差在 1 个 Tailwind 档位内）
- [ ] 配色提取合理（即使不完全匹配，也应接近）
- [ ] 阴影、间距等参数符合截图风格

✅ **错误处理**
- [ ] 无图片时点击按钮有明确错误提示
- [ ] 网络错误时有友好提示
- [ ] AI 返回格式异常时能优雅降级

✅ **用户体验**
- [ ] 提取结果清晰展示，用户能看懂
- [ ] 提取成功后有明确的成功提示
- [ ] 历史记录、描述功能不受影响

## 已知限制

1. **Tailwind 类名推断**
   - AI 推断的类名可能与实际使用的不完全一致
   - 例如：截图中可能是 `rounded-2xl`，AI 可能返回 `rounded-xl`

2. **复杂渐变与自定义样式**
   - 自定义 CSS 值（非 Tailwind 标准类）可能无法准确提取
   - 建议手动调整

3. **响应式断点**
   - 当前仅提取整体风格，不区分 `sm:`、`md:` 等响应式变体
