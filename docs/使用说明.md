# 使用说明

本工具用于管理 UI 审美参数，并自动生成可用于 Cursor 的提示词（Prompt）。

## 1. 启动项目

在项目根目录执行：

```
npm install
npm run dev
```

终端会显示本地访问地址，例如：

```
http://localhost:5176/
```

用浏览器打开该地址即可访问页面。

## 2. 配置参数

页面左侧是参数配置区，按 7 个维度组织：

- 容器 & 布局
- 圆角 (Radius)
- 背景 (Glass)
- 阴影 (Shadow)
- 字体 (Typo)
- 交互 (Motion)
- 色彩 (Color)

每个维度下的配置项包含：

- 组件名称
- CSS 类名（Tailwind）
- Prompt 片段

修改任意字段会实时更新右侧的 Prompt。

## 3. 生成与复制 Prompt

页面右侧会实时显示完整的 Prompt。

点击“复制”按钮可一键复制到剪贴板，直接粘贴到 Cursor 即可。

## 4. 配置持久化

所有配置会自动保存到浏览器的 `localStorage`。

再次打开页面时会自动加载上次的配置内容，无需手动保存。

## 5. 访问局域网

如果希望在局域网内的其他设备访问：

1. 确保开发服务器正在运行：`npm run dev`
2. 终端会显示 `Network` 地址（例如 `http://192.168.x.x:5176/`）
3. 用同一局域网的设备访问该地址即可

## 6. 常见问题

### 无法访问页面 / 连接被拒绝

检查终端输出的端口是否变化，确保使用最新的 `localhost:517x` 地址。

### 页面空白或报错

打开浏览器控制台（F12 → Console）查看错误信息，并提供给开发者定位问题。

## 7. AI 功能使用（图像理解）

本功能通过后端服务调用火山引擎模型，不会在前端暴露密钥。

### 配置步骤

1. 在系统环境变量或 `.env` 文件中设置：

```
ARK_API_KEY=你的API_KEY          # 必填：用于调用豆包模型的 API Key
AI_SERVER_PORT=5180              # 可选：后端服务端口，不配置时默认为 5180
ARK_MODEL_ID=doubao-seed-1-6-251015 # 可选：豆包模型 ID，建议使用后端代码中默认值或文档推荐值
```

2. 启动后端服务：

```
npm run dev:server
```

默认端口为 `5180`，前端会通过 `/api` 代理访问。

### 使用方式

1. 在页面右侧 "AI 图像理解" 卡片中输入图片 URL 或上传本地图片
2. 输入提问文本（默认"你看见了什么？"）
3. 点击"生成描述"，即可看到返回内容
4. 历史记录会自动保存，点击任意条目可快速回填

### 🎨 视觉反向工程（一键参数回填）

**新功能**：上传任意 UI 截图，AI 自动提取样式参数并回填到左侧配置表单！

#### 使用步骤

1. 在 "AI 图像理解" 卡片中上传你喜欢的 UI 截图（或输入图片 URL）
2. 点击 **"✨ 提取参数"** 按钮（绿色按钮）
3. AI 会自动分析截图中的：
   - 圆角大小（卡片、按钮、输入框）
   - 间距设置（内边距、元素间距）
   - 配色方案（主色、背景色、文字透明度）
   - 阴影效果（卡片阴影、按钮阴影）
   - 背景效果（毛玻璃）
   - 字体样式（标题、正文）
   - 交互动画（悬停、点击）
4. 提取成功后，左侧配置会自动更新，右侧 Prompt 也会同步变化
5. 你可以在自动填充的基础上继续微调

#### 提示

- 建议上传**清晰、完整**的 UI 截图，包含多种元素（按钮、卡片、输入框等）
- AI 会尽可能推断 Tailwind 类名，无法判断的参数会给出合理推荐值
- 如果提取结果不理想，可以手动调整左侧配置

### 请求体大小与图片体积建议

- 前端上传本地图片时，会将图片转为 base64 Data URL，通过 JSON 发送到后端  
- 后端默认的 JSON 请求体大小限制为 **20MB**，如果图片体积过大，可能会收到：
  - 前端提示：`请求体过大，请尝试使用更小的图片（或压缩后上传），或改用图片 URL 方式`
  - 或浏览器中返回 413 错误

**建议：**

- 优先使用公网图片 URL 方式调用（体积更小，传输更快）
- 如果必须上传本地图片，尽量控制在 **10MB 以下**，必要时先进行压缩
- 如需放宽限制，可在后端 `server/index.js` 中调整：
  - `express.json({ limit: '20mb' })` 中的 `limit` 值，例如改为 `30mb`、`50mb`

### 自定义端口与模型 ID

- 通过环境变量或 `.env` 文件可以配置：
  - `AI_SERVER_PORT`：后端监听端口，前端通过 `/api` 代理访问
  - `ARK_MODEL_ID`：豆包模型 ID，后端会优先读取环境变量，未设置时使用内置默认值

> 修改端口或模型 ID 后，记得重启后端服务以生效。

## 8. Python 图像理解脚本使用方法（本地命令行）

如果你更习惯在命令行中调用图像理解功能，可以使用项目根目录下的 `vision_demo.py` 脚本。  
**注意：该脚本不会直接持有 `ARK_API_KEY`，而是通过本地后端服务转发调用，由后端统一管理密钥。**

### 8.1 安装依赖

请确保本地已安装 Python 3.9+，然后在任意目录执行：

```bash
pip install requests
```

> 说明：本脚本仅通过 HTTP 调用你本机的后端服务，不直接调用火山引擎 SDK。

### 8.2 后端服务与环境变量

脚本不会读取 `ARK_API_KEY`，密钥只需要在后端服务中配置即可（与前端共用同一后端）。

1. 在系统环境变量中为后端配置：

```powershell
ARK_API_KEY=你的API_KEY
```

2. 启动后端服务（项目根目录）：

```powershell
npm run dev:server
```

默认地址为 `http://localhost:5180`，前端和 Python 脚本都会调用该服务。

3. 可选：如果后端运行在其他地址/端口（例如你映射到了局域网 IP），可以为脚本设置：

```powershell
$env:AI_SERVER_URL="http://localhost:5180"
```

脚本会优先读取 `AI_SERVER_URL`，也可以通过命令行参数 `--server-url` 显式指定。

### 8.3 运行脚本

在项目根目录或你复制出脚本所在的目录中执行，例如：

```powershell
python vision_demo.py --image-url "https://lf-sourcecdn-tos.bytegecko.com/obj/ark-train-demo/ark-demo-img-1.png" --question "请用中文详细描述这张图里你看到了什么？"
```

参数说明：

- `--image-url`：要分析的图片公网 URL，不指定时会使用脚本内置的示例图片
- `--question`：你的提问内容，不指定时会使用脚本内置的示例问题
- `--server-url`：后端服务地址，默认为 `http://localhost:5180`，也可以通过环境变量 `AI_SERVER_URL` 配置

### 8.4 推荐使用流程

1. 在浏览器中打开本 UI 工具，按 7 个维度配置 UI 参数，生成并复制 Prompt（与现在用法完全一致）
2. 根据需要准备一张需要理解的图片（例如文档中同一张官方示例图 `ark_demo_img_1.png`）
3. 在命令行中调用：

```powershell
python vision_demo.py --image-url "https://lf-sourcecdn-tos.bytegecko.com/obj/ark-train-demo/ark-demo-img-1.png" --question "请用中文详细描述这张图里你看到了什么？"
```

4. 在终端输出中查看模型返回的中文描述结果

### 8.5 安全与注意事项

- **不要**把 `ARK_API_KEY` 写死在任何前端代码或脚本中，也不要提交到公共仓库，统一由后端持有和管理
- 建议仅在本机或受信任的内网环境中使用该脚本与后端服务
- 如需做成通用 CLI 工具，可以在本脚本基础上扩展参数解析与错误处理逻辑，本项目中保持当前最小可用实现
