## ⚙️ 服务器端配置

### 1. 确保 Docker PostgreSQL 运行

SSH 登录服务器：

```bash
ssh user@107.173.156.235
```

检查 PostgreSQL 容器状态：

```bash
# 查看运行中的容器
docker ps | grep postgres

# 如果容器未运行，启动容器
docker start <container_name_or_id>
```

### 2. 确保端口映射正确

检查 Docker 容器端口映射：

```bash
docker port <container_name_or_id>
```

**预期输出：**
```
5432/tcp -> 0.0.0.0:5432
```

如果端口未映射，重新创建容器：

```bash
docker run -d \
  --name postgres-ui-tools \
  -e POSTGRES_PASSWORD=你的密码 \
  -p 5432:5432 \
  -v pgdata:/var/lib/postgresql/data \
  postgres:15
```

### 3. 配置防火墙

确保服务器防火墙允许 5432 端口：

```bash
# Ubuntu/Debian
sudo ufw allow 5432/tcp
sudo ufw reload

# CentOS/RHEL
sudo firewall-cmd --add-port=5432/tcp --permanent
sudo firewall-cmd --reload
```

### 4. 配置 PostgreSQL 允许远程连接

进入容器：

```bash
docker exec -it <container_name> bash
```

编辑 `postgresql.conf`：

```bash
# 在容器内执行
cd /var/lib/postgresql/data
vi postgresql.conf

# 找到并修改：
listen_addresses = '*'
```

编辑 `pg_hba.conf`：

```bash
vi pg_hba.conf

# 在文件末尾添加：
host    all             all             0.0.0.0/0               md5
```

重启容器：

```bash
docker restart <container_name>
```

### 5. 创建数据库

连接到 PostgreSQL：

```bash
# 在服务器上执行
docker exec -it <container_name> psql -U postgres

# 在 psql 中执行
CREATE DATABASE ui_tools;
\q
```

## 🔒 安全建议

### 1. 修改默认密码

```bash
docker exec -it <container_name> psql -U postgres

# 在 psql 中执行
ALTER USER postgres WITH PASSWORD '强密码';
\q
```

### 2. 创建专用用户

```bash
docker exec -it <container_name> psql -U postgres

# 在 psql 中执行
CREATE USER ui_tools_user WITH PASSWORD '强密码';
CREATE DATABASE ui_tools OWNER ui_tools_user;
GRANT ALL PRIVILEGES ON DATABASE ui_tools TO ui_tools_user;
\q
```

然后在 `.env` 中使用专用用户：

```env
POSTGRES_USER=ui_tools_user
POSTGRES_PASSWORD=强密码
```

### 3. 限制访问来源

修改 `pg_hba.conf`，只允许特定 IP：

```bash
# 替换 0.0.0.0/0 为你的 IP
host    ui_tools    ui_tools_user    你的IP/32    md5
```

### 4. 使用 SSL 连接（可选）

在 `server/db.js` 中启用 SSL：

```javascript
const pool = new Pool({
  host: process.env.POSTGRES_HOST,
  port: parseInt(process.env.POSTGRES_PORT || '5432'),
  database: process.env.POSTGRES_DB,
  user: process.env.POSTGRES_USER,
  password: process.env.POSTGRES_PASSWORD,
  ssl: {
    rejectUnauthorized: false  // 开发环境
    // 生产环境应使用：
    // rejectUnauthorized: true,
    // ca: fs.readFileSync('/path/to/server-certificates/root.crt').toString(),
  },
  max: 20,
});
```

## ❓ 常见问题

### Q: 连接超时

**错误：`connection timed out`**

**原因：**
1. 服务器防火墙阻止了 5432 端口
2. PostgreSQL 容器未启动
3. 网络连接问题

**解决：**
```bash
# 1. 检查容器状态
docker ps | grep postgres

# 2. 测试端口连通性（在本地执行）
telnet 107.173.156.235 5432
# 或使用 nc
nc -zv 107.173.156.235 5432

# 3. 检查防火墙
sudo ufw status
```

### Q: 密码认证失败

**错误：`password authentication failed`**

**解决：**
1. 确认 `.env` 中的密码正确
2. 检查 `pg_hba.conf` 配置
3. 尝试重置密码

### Q: 数据库不存在

**错误：`database "ui_tools" does not exist`**

**解决：**
```bash
docker exec -it <container_name> psql -U postgres -c "CREATE DATABASE ui_tools;"
```

### Q: 权限不足

**错误：`permission denied`**

**解决：**
```bash
docker exec -it <container_name> psql -U postgres

# 在 psql 中执行
GRANT ALL PRIVILEGES ON DATABASE ui_tools TO postgres;
\q
```

## 🧪 测试连接

创建测试脚本 `test-db-connection.js`：

```javascript
import pg from 'pg';
import dotenv from 'dotenv';

dotenv.config();

const { Pool } = pg;

const pool = new Pool({
  host: process.env.POSTGRES_HOST,
  port: parseInt(process.env.POSTGRES_PORT || '5432'),
  database: process.env.POSTGRES_DB,
  user: process.env.POSTGRES_USER,
  password: process.env.POSTGRES_PASSWORD,
});

async function testConnection() {
  try {
    console.log('🔗 正在连接数据库...');
    console.log(`   主机: ${process.env.POSTGRES_HOST}`);
    console.log(`   端口: ${process.env.POSTGRES_PORT}`);
    console.log(`   数据库: ${process.env.POSTGRES_DB}`);
    console.log(`   用户: ${process.env.POSTGRES_USER}`);
    
    const client = await pool.connect();
    console.log('✅ 连接成功！');
    
    const result = await client.query('SELECT version()');
    console.log('📊 PostgreSQL 版本:', result.rows[0].version);
    
    client.release();
    await pool.end();
    
    console.log('✅ 测试完成！');
  } catch (error) {
    console.error('❌ 连接失败:', error.message);
    process.exit(1);
  }
}

testConnection();
```

运行测试：

```bash
node test-db-connection.js
```

## 📊 监控与维护

### 查看数据库大小

```bash
docker exec -it <container_name> psql -U postgres -d ui_tools

# 在 psql 中执行
\l+  # 查看所有数据库大小
\dt+ # 查看所有表大小
```

### 备份数据库

```bash
# 导出备份
docker exec -t <container_name> pg_dump -U postgres ui_tools > ui_tools_backup_$(date +%Y%m%d).sql

# 恢复备份
docker exec -i <container_name> psql -U postgres ui_tools < ui_tools_backup_20260205.sql
```

### 查看当前连接

```bash
docker exec -it <container_name> psql -U postgres -d ui_tools

# 在 psql 中执行
SELECT * FROM pg_stat_activity WHERE datname = 'ui_tools';
```

### Docker Compose 推荐配置

如果你想在服务器上使用 Docker Compose 管理 PostgreSQL：

```yaml
version: '3.8'

services:
  postgres:
    image: postgres:15
    container_name: postgres-ui-tools
    environment:
      POSTGRES_PASSWORD: 你的密码
      POSTGRES_DB: ui_tools
    ports:
      - "5432:5432"
    volumes:
      - pgdata:/var/lib/postgresql/data
    restart: unless-stopped
    networks:
      - ui-tools-network

volumes:
  pgdata:

networks:
  ui-tools-network:
    driver: bridge
```

启动：
```bash
docker-compose up -d
```
