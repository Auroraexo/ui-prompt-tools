# PostgreSQL 数据库配置指南

## 🎯 功能说明

配置管理功能允许你：
- ✅ 将 UI 参数配置保存到 PostgreSQL 数据库
- ✅ 多设备同步（所有连接同一数据库的设备共享配置）
- ✅ 团队协作（团队成员可以加载彼此保存的配置方案）
- ✅ 配置版本管理（保存多个不同的 UI 风格方案）
- ✅ 历史记录追溯（每个配置自动记录创建和更新时间）

## 📋 前提条件

### 1. 安装 PostgreSQL

#### Windows
1. 下载安装包：[PostgreSQL 官网](https://www.postgresql.org/download/windows/)
2. 运行安装程序，按默认选项安装
3. 记住安装时设置的 `postgres` 用户密码

#### macOS
```bash
# 使用 Homebrew 安装
brew install postgresql@15
brew services start postgresql@15
```

#### Linux (Ubuntu/Debian)
```bash
sudo apt update
sudo apt install postgresql postgresql-contrib
sudo systemctl start postgresql
```

### 2. 验证 PostgreSQL 安装

```bash
# 查看 PostgreSQL 版本
psql --version

# 输出示例：psql (PostgreSQL) 15.x
```

## 🚀 快速开始

### 步骤 1：创建数据库

```bash
# Windows: 打开 SQL Shell (psql)
# macOS/Linux: 打开终端

# 登录 PostgreSQL（会提示输入密码）
psql -U postgres

# 在 psql 中执行以下命令：
CREATE DATABASE ui_tools;

# 退出 psql
\q
```

### 步骤 2：配置环境变量

复制 `env.example` 为 `.env`，然后编辑 `.env` 文件：

```env
# PostgreSQL 数据库配置
POSTGRES_HOST=localhost
POSTGRES_PORT=5432
POSTGRES_DB=ui_tools
POSTGRES_USER=postgres
POSTGRES_PASSWORD=你的数据库密码
```

### 步骤 3：安装依赖

```bash
npm install
```

### 步骤 4：初始化数据库表

```bash
npm run db:init
```

**预期输出：**
```
[DB] 开始初始化数据库...
[DB] ✅ 表 ui_presets 已创建或已存在
[DB] ✅ 索引已创建
[DB] 当前配置数量: 0
[DB] ✅ 数据库初始化完成！
```

### 步骤 5：启动服务

```bash
# 一键启动前后端
npm run dev:all

# 或分别启动
npm run dev:server  # 终端 1
npm run dev         # 终端 2
```

## 📖 使用说明

### 保存配置

1. 在左侧调整好你的 UI 参数
2. 点击右侧顶部的 **"💾 保存当前配置"** 按钮
3. 在弹窗中输入：
   - **配置名称**（必填）：例如"现代毛玻璃风格"
   - **描述**（可选）：简要说明这个配置的特点
4. 点击"确认保存"

### 加载配置

1. 在 **"已保存的配置"** 列表中找到目标配置
2. 点击 **"加载"** 按钮
3. 左侧配置会自动更新为该配置的参数
4. 右侧 Prompt 同步更新

### 删除配置

1. 点击配置右侧的 **"删除"** 按钮
2. 确认删除操作

## 🗄️ 数据库表结构

```sql
CREATE TABLE ui_presets (
  id SERIAL PRIMARY KEY,
  preset_name VARCHAR(255) NOT NULL,
  description TEXT,
  config_data JSONB NOT NULL,        -- 核心：使用 JSONB 存储配置
  full_prompt TEXT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 索引优化
CREATE INDEX idx_preset_name ON ui_presets(preset_name);
CREATE INDEX idx_config_data ON ui_presets USING GIN (config_data);
```

### 字段说明

| 字段 | 类型 | 说明 |
|-----|------|------|
| `id` | SERIAL | 主键，自增 |
| `preset_name` | VARCHAR(255) | 配置名称 |
| `description` | TEXT | 配置描述（可选） |
| `config_data` | JSONB | **核心字段**：存储完整的配置数据 |
| `full_prompt` | TEXT | 自动生成的完整 Prompt |
| `created_at` | TIMESTAMP | 创建时间 |
| `updated_at` | TIMESTAMP | 更新时间 |

## 🔧 API 接口

## ❓ 常见问题

### Q: 数据库初始化失败怎么办？

**错误 1：`FATAL: password authentication failed`**
- 原因：数据库密码错误
- 解决：检查 `.env` 中的 `POSTGRES_PASSWORD` 是否正确

**错误 2：`FATAL: database "ui_tools" does not exist`**
- 原因：数据库尚未创建
- 解决：执行 `CREATE DATABASE ui_tools;`

**错误 3：`connection refused`**
- 原因：PostgreSQL 服务未启动
- 解决：启动 PostgreSQL 服务
  - Windows: 打开"服务"，启动 `postgresql-x64-15`
  - macOS: `brew services start postgresql@15`
  - Linux: `sudo systemctl start postgresql`

### Q: 如何查看数据库中的数据？

```bash
# 登录数据库
psql -U postgres -d ui_tools

# 查看所有配置
SELECT id, preset_name, created_at FROM ui_presets;

# 查看单个配置的完整数据
SELECT * FROM ui_presets WHERE id = 1;
```

### Q: 如何备份配置？

```bash
# 导出整个数据库
pg_dump -U postgres ui_tools > ui_tools_backup.sql

# 恢复备份
psql -U postgres ui_tools < ui_tools_backup.sql
```

### Q: 如何清空所有配置？

```bash
psql -U postgres -d ui_tools

# 删除所有配置
DELETE FROM ui_presets;

# 重置自增 ID
ALTER SEQUENCE ui_presets_id_seq RESTART WITH 1;
```

## 🎯 进阶使用

### 团队协作

1. **搭建共享数据库**
   - 在云服务器（阿里云/腾讯云）上安装 PostgreSQL
   - 配置防火墙允许团队成员访问

2. **团队成员配置**
   - 每个成员的 `.env` 文件中配置相同的数据库地址
   - 共享数据库凭证（注意安全）

3. **协作流程**
   - 设计师保存多个风格方案
   - 开发者加载对应方案生成代码
   - 所有人看到相同的配置列表

### JSONB 高级查询

PostgreSQL 的 JSONB 字段支持复杂查询：

```sql
-- 查找所有使用"rounded-3xl"圆角的配置
SELECT preset_name 
FROM ui_presets 
WHERE config_data -> '圆角 (Radius)' @> '[{"cssClass": "rounded-3xl"}]';

-- 查找所有使用"indigo"主色的配置
SELECT preset_name 
FROM ui_presets 
WHERE config_data::text LIKE '%indigo%';
```
