# 🚀 快速启动指南

## 一键启动（推荐）

如果你已经安装了依赖并配置了环境变量，只需运行：

```bash
npm run dev:all
```

这个命令会同时启动前端和后端服务。

## 分步启动

### 第一次使用

#### 1. 安装依赖
```bash
npm install
```

#### 2. 配置环境变量
复制 `env.example` 为 `.env`，并填入你的 API Key：

```bash
# Windows PowerShell
copy env.example .env

# macOS/Linux
cp env.example .env
```

然后编辑 `.env` 文件：
```
ARK_API_KEY=你的火山引擎豆包API密钥
```

#### 3. 启动服务

**方式 1：一键启动（推荐）**
```bash
npm run dev:all
```

**方式 2：分别启动**
```bash
# 终端 1：启动后端
npm run dev:server

# 终端 2：启动前端
npm run dev
```

#### 4. 访问应用

浏览器会自动打开，或手动访问终端显示的地址（通常是 `http://localhost:5176`）。

## 常见问题

### ❌ 启动失败：缺少 ARK_API_KEY

**原因**：未配置环境变量

**解决**：
1. 确保 `.env` 文件存在于项目根目录
2. 确保 `ARK_API_KEY=你的密钥` 格式正确（等号两边无空格）
3. 重启服务

### ❌ 端口冲突：5180 或 5176 被占用

**解决**：
```bash
# 方式 1：在 .env 中修改端口
AI_SERVER_PORT=5181

# 方式 2：关闭占用端口的进程
# Windows
netstat -ano | findstr :5180
taskkill /PID <进程ID> /F

# macOS/Linux
lsof -ti:5180 | xargs kill -9
```

### ❌ 提取参数失败：请求超时

**原因**：网络不稳定或图片过大

**解决**：
1. 检查网络连接
2. 压缩图片到 2MB 以下
3. 改用公网图片 URL（而非本地上传）

### ❌ 上传失败：请求体过大

**解决**：
1. 压缩图片（推荐 < 2MB）
2. 或修改 `server/index.js` 中的 `limit: '20mb'` 为更大值（如 `'50mb'`）
