# 🚀 快速启动指南（数据库版）

本指南适用于需要配置持久化功能（PostgreSQL 数据库）的用户。

## ⚡ 一键安装脚本（推荐）

### Windows PowerShell

```powershell
# 1. 安装依赖
npm install

# 2. 配置环境变量（复制并编辑 .env 文件）
Copy-Item env.example .env
# 然后用记事本打开 .env 填入：
#   - ARK_API_KEY
#   - POSTGRES_PASSWORD

# 3. 创建数据库（需要先安装 PostgreSQL）
psql -U postgres -c "CREATE DATABASE ui_tools;"

# 4. 初始化数据库表
npm run db:init

# 5. 一键启动
npm run dev:all
```

### macOS / Linux

```bash
# 1. 安装依赖
npm install

# 2. 配置环境变量
cp env.example .env
# 然后用 nano/vim 编辑 .env 填入必要配置

# 3. 创建数据库
psql -U postgres -c "CREATE DATABASE ui_tools;"

# 4. 初始化数据库表
npm run db:init

# 5. 一键启动
npm run dev:all
```

## 📋 详细步骤

### 步骤 1：安装 PostgreSQL

#### Windows
下载并安装：[PostgreSQL 官网](https://www.postgresql.org/download/windows/)

#### macOS
```bash
brew install postgresql@15
brew services start postgresql@15
```

#### Linux (Ubuntu)
```bash
sudo apt update && sudo apt install postgresql postgresql-contrib
sudo systemctl start postgresql
```

### 步骤 2：创建数据库

**注意：你的数据库在远程服务器 `107.173.156.235`，请在服务器上执行：**

```bash
# SSH 登录服务器
ssh user@107.173.156.235

# 进入 PostgreSQL Docker 容器
docker exec -it <container_name> psql -U postgres

# 创建数据库
CREATE DATABASE ui_tools;

# 退出
\q
```

**或者直接从服务器执行：**
```bash
docker exec -it <container_name> psql -U postgres -c "CREATE DATABASE ui_tools;"
```

### 步骤 3：配置项目

```bash
# 1. 安装依赖
npm install

# 2. 复制环境变量模板
cp env.example .env  # Windows: copy env.example .env

# 3. 编辑 .env 文件，填入以下必要配置：
```

`.env` 文件内容：
```env
ARK_API_KEY=你的火山引擎API密钥

# 远程数据库配置
POSTGRES_HOST=107.173.156.235
POSTGRES_PORT=5432
POSTGRES_DB=ui_tools
POSTGRES_USER=postgres
POSTGRES_PASSWORD=你的数据库密码
```

### 步骤 4：初始化数据库表

```bash
npm run db:init
```

**预期输出：**
```
[DB] 开始初始化数据库...
[DB] ✅ 表 ui_presets 已创建或已存在
[DB] ✅ 索引已创建
[DB] 当前配置数量: 0
[DB] ✅ 数据库初始化完成！
```

### 步骤 5：启动服务

```bash
npm run dev:all
```

访问：`http://localhost:5176`

## ❓ 常见问题

### Q: 数据库初始化失败

**错误：`FATAL: database "ui_tools" does not exist`**
- 解决：执行 `psql -U postgres -c "CREATE DATABASE ui_tools;"`

**错误：`password authentication failed`**
- 解决：检查 `.env` 中的 `POSTGRES_PASSWORD` 是否正确

**错误：`connection refused`**
- 解决：确保 PostgreSQL 服务已启动
  - Windows: 打开"服务"，启动 `postgresql-x64-15`
  - macOS: `brew services start postgresql@15`
  - Linux: `sudo systemctl start postgresql`

### Q: 端口被占用

**错误：`Port 5180 is already in use`**
- 解决：修改 `.env` 中的 `AI_SERVER_PORT=5181`

**错误：`Port 5176 is already in use`（前端）**
- 解决：Vite 会自动切换到下一个可用端口（5177）

### Q: 无法保存配置

- 检查后端终端是否有数据库连接错误
- 检查浏览器控制台（F12）是否有请求失败
- 确认 PostgreSQL 服务正在运行
